APICloud 认证考试（9月15日）说明文档

1. 考试要求及说明

  • 确保自己电脑及网络可以正常使用
  • 进入官网，登录报名的邮箱账号->系统会自动分配考试应用，应用名称为“日期+姓名”->请在分配的应用下完成考试
  • 考试时间为3个小时，考试时间为14:00-17:00，请在17:00前准时提交代码，超出时间与往期一样成绩为零分，监考老师会提前15分钟提醒大家；（如有特殊情况请提前找监考老师备注信息）
  • 监考老师会在考试时间内不定时的会抽查考试同学的情况，如果发现作弊，雷同代码，将取消考试成绩，并取消免费补考资格
  • 考试时间内，会给大家安排考试技术支持，遇到问题可以在私聊QQ群中的技术支持，技术支持会帮助您解答遇到的问题
  • 考试成绩会在30天内公布，成绩单（考试得分等级和评语）会发送至报名邮箱，通过考试的在个人账户信息ace证书置亮，开通认证专区（ps成绩单发布的邮箱如果和考试报名邮箱不一致的小伙伴，提前qq3266304342联系备注成绩单发至的邮箱）

  • 如何交卷：考生需要在17:00点,把代码通过APIClodu Stuido 或第三方SVN工具 ,把代码提交到云端.17:00 后提交的代码,不计算成绩. 使用第三方SVN工具时,需要的SVN地址和密码,可以在应用云端控制台 —> 代码 界面获取.（17:00点提交后，就不要在操作更新代码，更新将会取消成绩）

    交卷方式:

    • APICloud Studio2 提交代码,步骤是: 右键项目根目录 --> git --> add + commit --> 在底部弹出框,输入一段描述文字,然后 cmd/ctrl + s保存 --> 右键项目根目录 --> 代码管理 --> 同步到云端
    • 可选使用第三方 SVN 工具提交代码,用户名和密码获取方式: 邮箱: 登录SVN工具的用户名为您在APICloud注册时的邮箱 密码: 网站控制台 --> 选择自己的应用 --> 选择 "代码" --> 选择 "获取分支密码"
    • 可选直接在把代码zip包上传,方式是: 网站控制台 --> 选择自己的应用 --> 选择 "代码" --> 选择 "上传代码"

2. 考试资料说明

  • 安装二维码——考试应用示范DEMO的安装二维码

  • UE——存放UE相关页面，考试App的交互逻辑可在此文件夹内查看，入口页面index.html或start.html;

    本UE文件已添加交互逻辑，考生可点击查看交互逻辑或考试应用示范DEMO进行交互逻辑的参考

  • UI——存放UI相关页面，在页面内可用鼠标测量相关页面元素的尺寸距离，入口页面为index.html;

  • UI->assets文件夹——存放项目使用的切图素材

  • UI->assets->loading文件夹——存放列表的下拉刷新使用的动画素材切图

  • 接口文档——服务器端接口文档，App与服务端请求的相关接口的定义描述

3. 评分标准

3.1. 页面UI：

  • 实现参考应用中全部界面;

  • 严格按照资源包中的UI图进行量图和布局;

  • 适配所有主流屏幕尺寸;

3.2. 功能逻辑：

应实现参考应用中全部功能

  • 主页面

    • 实现【测试页】和【记录页】的滑动切换
    • 实现android的3秒内双击keyback键退出应用功能

  • 主页面-测试页

    • 实现输入昵称、选择性别后，点击“开始答题”按钮，进入【答题详情页】
    • 实现必要的参数逻辑校验

  • 主页面-记录页

    • 列表应实现下拉刷新、上拉加载功能
    • 下拉刷新应使用模块UIPush模块实现动画效果
    • 点击列表单元格，可跳转“性格详情页”查看对应的性格详情

  • 测试详情页

    • 实现题目页面的滑动切换
    • 在第一题是，应将“前一题”按钮置灰；最后一题应将“后一题”按钮动态切换为“提交”按钮

  • 性格详情页

    调用服务端的H5页面进行展示

3.3. 性能体验：

  • 交互响应灵敏
  • 界面切换流畅
  • 无闪屏白屏卡顿等
  • 按钮点击优化
  • 是否存在Bug

4.数据交互：

  • 使用 api.ajax 进行数据通信
  • 实现下拉刷新（使用UIPullRefresh模块），上拉加载更多（分页查询）

5.代码质量：

检查称编码是否遵循APICloud App开发规范，包括但不限于下列内容

  • 命名规范
  • 代码注释
  • 代码逻辑
  • 代码整洁
  • 异常处理
  • 代码封装
